## Инструкция по структуре коммит-месседжей

### Формат:
```
<type>(<scope>): <subject>
```

---

### 1. Type (тип изменения):

Всегда на **английском**, один из:

- **feat** — новая функциональность
- **fix** — исправление бага
- **docs** — изменения в документации
- **style** — форматирование кода (без изменения логики)
- **refactor** — рефакторинг без изменения функциональности
- **perf** — улучшение производительности
- **test** — добавление или исправление тестов
- **build** — изменения системы сборки или зависимостей
- **ci** — изменения CI/CD конфигурации
- **chore** — прочие изменения (обновление зависимостей, настройки и т.д.)

---

### 2. Scope (область изменения):

На **английском**, один из:
- Название внутреннего пакета (например: `server`, `web`, `youtube-authenticator`)
- Репозиторий в целом (`repo`)

---

### 3. Subject (описание):

На **русском языке**, следуя правилам:
- ✅ Повелительное наклонение: "добавить", "исправить", "обновить"
- ✅ Без заглавной буквы в начале
- ✅ Без точки в конце
- ✅ Максимум 72 символа
- ✅ Четко и кратко описывает ЧТО сделано

---

### Примеры:

```bash
feat(speech-api): добавить интеграцию с SaluteSpeech
fix(ydb-connector): исправить обработку timeout соединения
docs(terraform): добавить примеры конфигурации YDB
refactor(message-queue): оптимизировать пакетную обработку
chore(deps): обновить TypeScript до версии 5.3
perf(api-gateway): уменьшить время холодного старта
test(lockbox): добавить тесты получения секретов
build(docker): обновить образ Node.js до 20.x
ci: настроить автоматический деплой на staging
style(api): привести код к единому стилю
```

### Примеры без scope:

```bash
chore: обновить зависимости
docs: исправить опечатки в README
ci: обновить workflow для деплоя
```

---

### ❌ Неправильно:

```bash
Added new feature                    # type отсутствует, на английском
fix: Исправлен баг.                 # заглавная буква, точка в конце
feat(api): добавил новый эндпоинт   # не повелительное наклонение
```

### ✅ Правильно:

```bash
feat(api): добавить новый эндпоинт
fix(auth): исправить проблему с токенами
docs: обновить документацию по API
```
