# Title
Изменить Storage на JSON

# Description
Переделать StorageService для работы с .json файлами. Task interface: { id: string, data: Record<string, unknown> }.

Q&A:
- JSON error details: Подробные (rawContent, position, syntax error)
- Validate before write: Да, throw StorageWriteError если non-serializable
- Required fields: Нет, любой Record<string, unknown> валиден
- Warning location: console.warn в stderr
- What counts as invalid: Только syntax errors (runtime errors throw)
- JSON.stringify timing: До temp file (fails fast)
- Validate after write: Нет, доверяем atomic rename
- Empty values: undefined → null,  →  (включать все)
- File locking: Нет, atomic rename достаточно
- Existing .md: Полный игнор (list фильтрует, exists=false, delete throws)
- .md.tmp files: Игнорировать полностью
- Test data: Минимальная структура { title: "Task" } для basic tests
- Test structure: Создавать JSON напрямую через vol.promises.writeFile
- read() return: { id, data } object (consistent with list)

read() → JSON object, create/update() → object + JSON.stringify(data, null, 2) внутри. Фильтр .json файлов. Ошибки JSON → StorageInvalidJsonError. Пустые поля ВКЛЮЧАТЬ. Graceful degradation — skip с warning. Существующие .md игнорировать.
