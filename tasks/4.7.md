# Title
Создать команду migrate

# Description
Создать команду migrate для конвертации одного .md файла в .json. Usage: hod migrate <file.md>

Q&A:
- File path resolution: Относительно cwd (любой валидный путь)
- Task ID extraction: Basename без расширения (path/to/1.md → ID=1)
- Validate in tasksDir: Нет, мигрировать можно любой .md файл
- JSON structure: Плоский объект { id, title, description, status, dependencies, customFields... }
- Custom field keys: kebab-case (из field.name)
- JSON location: Там же где .md файл
- Write mechanism: Прямой fs.writeFile (обход Storage, т.к. тот ещё .json не поддерживает)
- ParseError (no Title): Fail с ошибкой exit code 1
- Not in index: Use defaults (status=pending, deps=[]) с warning
- File not found: Fail с exit code 1
- JSON exists: Перезаписывать silently
- Partial failures: Не оставлять artifacts (clean up temp)

Принимает путь к .md, читает через Parser.parse(), status/deps из index (defaults: pending, []), создаёт JSON с ВСЕМИ полями включая пустые, сохраняет как .json рядом. Оригинал .md НЕ удаляет. Success: '✓ Migrated X.md → X.json'. Exit code 1 при ошибке.
